Python and Raspberry Pi

I tried out Python programming with the Raspberry Pi. This is really convenient because one of my goals is to create some interactive media displays. In my current job at the public library, I spend Sunday mornings learning about technologies I’m less familiar with, so this was prime. I followed this tutorial. Here’s the output:

I got the board out. Yay! I tried hooking up the LED and resister to the 3.3V and ground pins. The LED didn’t light up. :-/ I tried switching the resisters out, but I think they were all too high a resistance. In the end I took them out. And the light finally came on! It didn’t explode on me while I was using it as the tutorial warned. I’ve had LEDs burn out on me but not explode.

At this point the instructions said to open IDLE to run Python commands. I opened IDLE on my Mac but couldn’t find anything related to Raspberry Pi in the IDE. After consulting my colleague George, I discovered that, unlike Arduino, you need to interact with the Pi through its GUI. I got stuck here trying to sort all that out while waiting on replies from George (who wasn’t working today).

Anyway, I’m glad to get started with it. I didn’t make it to the programming part today, but that’s okay. I didn’t know what to expect until I started messing with it. I’ve been learning Python in Treehouse and I’m hoping combining it with Raspberry Pi will give me some good options for creating interactive projects.

IoT with ESP8266

The ESP8266 is an inexpensive WiFi enabled microcontroller. It's useful for IoT projects. At Make Santa Fe during an IoY workshop, I learned how to program one to send messages after sensing an input and to produce and output from Blynk, an app for creating interfaces for IoT objects.
